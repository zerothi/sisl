sisl.viz.FigureÔÉÅ

class sisl.viz.Figure[source]ÔÉÅ

Bases: object

Base figure class that all backends should inherit from.

It contains all the plotting actions that should be supported by a figure.

A subclass for a specific backend should implement as many methods as possible from the ones where Figure returns NotImplementedError. Other methods are optional because Figure contains a default implementation using other methods, which should work for most backends.

To create a new backend, one might take the PlotlyFigure as a template.

Same as draw line, but to draw a line with an area. This is for example used to draw fatbands.

Parameters:

  • x (ndarray) ‚Äì the coordinates of the points along the X axis.

  • y (ndarray) ‚Äì the coordinates of the points along the Y axis.

  • name (str, optional) ‚Äì the name of the scatter

  • line (dict, optional) ‚Äì specifications for the line style, following plotly standards. The backend should at least be able to implement line[‚Äúcolor‚Äù] and line[‚Äúwidth‚Äù], but it is very advisable that it supports also line[‚Äúopacity‚Äù].

  • text (str, optional) ‚Äì contains the text asigned to each marker. On plotly this is seen on hover, other options could be annotating. However, it is not necessary that this argument is supported.

  • dependent_axis (str, optional) ‚Äì The axis that contains the dependent variable. This is important because the area is drawn in parallel to that axis.

  • row (int, optional) ‚Äì If the figure contains subplots, the row where to draw.

  • col (int, optional) ‚Äì If the figure contains subplots, the column where to draw.

  • **kwargs ‚Äì should allow other keyword arguments to be passed directly to the creation of the scatter. This will of course be framework specific

Parameters:

  • xy (ndarray of shape (n_arrows, 2)) ‚Äì the positions where the atoms start.

  • dxy (ndarray of shape (n_arrows, 2)) ‚Äì the arrow vector.

  • arrow_head_scale (float, optional) ‚Äì how big is the arrow head in comparison to the arrow vector.

  • arrowhead_angle (angle) ‚Äì the angle that the arrow head forms with the direction of the arrow (in degrees).

  • scale (float, optional) ‚Äì multiplying factor to display the arrows. It does not affect the underlying data, therefore if the data is somehow displayed it should be without the scale factor.

  • annotate (bool) ‚Äì whether to annotate the arrows with the vector they represent.

  • row (int, optional) ‚Äì If the figure contains subplots, the row where to draw.

  • col (int, optional) ‚Äì If the figure contains subplots, the column where to draw.
